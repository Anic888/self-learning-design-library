# Ghost in the Shell (2017) — UI / Motion Graphics Design

**Designer:** Ash Thorp (with collaborative motion team)
**Year:** 2017
**Source URL:** ashthorp.com (case study) / Behance portfolio
**Project type:** Live-action film UI/motion graphics design
**Client:** Paramount Pictures / Rupert Sanders (director)

## Brief
UI and motion-graphics design for the 2017 live-action *Ghost in the Shell* film. Visual language for Section 9's tactical displays, thermoptic camouflage overlays, cybernetic augmentation readouts, and the Major's in-helmet HUD. Design must feel futuristic but grounded — rejecting both chrome/glossy sci-fi cliché AND the original anime's unique aesthetic.

## Palette
- `#00F0FF` / `#22D9FF` — cyan (primary interface color, 40-50%)
- `#06090D` / `#000810` — deep near-black base (30-40%)
- `#F2F2F2` — off-white (hairline linework, text, 10-15%)
- `#FF2E3A` / `#E01930` — red (alerts, targeting, threat indicators only, 3-5%)
- `#FF6A00` — amber (rare warmth element)

**Distribution rule:** cyan dominates, red is reserved exclusively for tension (weapons-lock, hostile detection, system breach). Off-white carries data. Black is context.

## Typography
- **Headlines / readouts:** Eurostile Extended Medium, ultra-wide tracking (200+)
- **Data:** condensed mono (Input Mono, or custom mono)
- **Kanji / katakana** embedded as tertiary layer (authentic Japanese script, not decorative nonsense)
- All caps throughout interface
- Light weight dominant — NEVER bold
- Mix of Latin + Japanese creates cultural depth unique to this project

## Composition (per-frame analysis)
- **Layered planes** — foreground UI (closest to eye), mid-range data panels, atmospheric backdrop
- **HUD elements clustered at edges** — corners and sides reserved for data, center for content/threat
- **Circular reticles / targeting** — concentric thin rings with tick marks
- **Data readouts scroll/animate** — information is alive, not static
- **Negative space dominant** in center — interface does NOT obscure the world being analyzed

## Techniques observed
- **Thin hairline strokes (0.3-0.8px)** throughout — zero heavy solid shapes
- **Subtle glow** on cyan lines (not overblown, just implied luminosity)
- **Motion principle: data always moving** — even at rest, subtle breathing/pulsing
- **Layered opacity** — UI has 3-4 depth planes suggested through transparency
- **Glitch/distortion** only at system-stress moments (never decorative)
- **Subtle scan-sweep** across circular reticles — suggests active scanning
- **Kanji appears/disappears** on layered planes — adds cultural/atmospheric depth without narrative significance

## Visual hierarchy (top to bottom importance)
1. Threat indicator (red when present) — immediately drawable
2. Target/subject of analysis (person, object, location)
3. Data readouts (identity, vital signs, location coordinates)
4. Reticle/framing system (tells viewer what's being analyzed)
5. Atmospheric backdrop (city, environment)

## What's stolen-worthy for other projects
- **Cyan + red + off-white + near-black** — tight 4-color palette that scales to any sci-fi UI context
- **Hairline strokes ONLY** — no solid decorative shapes anywhere
- **Concentric thin rings as targeting/reticle** — reusable pattern for cyberpunk UI compositions
- **Kanji/katakana as atmospheric layer** — adds cultural depth without requiring translation comprehension
- **Wide-tracked light-weight uppercase** — the "legibly serious" typography move
- **HUD elements clustered at edges, center reserved** — composition rule for interface over content

## Caveats (what's NOT easily reproducible)
- Motion is essential — static screenshots lose 50% of the language
- Requires AE / motion expertise to execute
- Film context gives this UI physical scale/drama — hard to replicate in print work
- Collaborative team output (Thorp directed but many hands contributed)
